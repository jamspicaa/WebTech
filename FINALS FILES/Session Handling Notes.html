Session Handling in Java Servlet

  • What is a Session?
    • Session is the communication between the client and the server that is compose of multiple requests and responses between client and server.
    • Session is being maitained if some information about the session is being passed between the server and client everytime that there is a responses and requests that is happening.
    • Ways to provide an identifier
      • User Authentication
        • User can give authentication credentials in order to pass authentication information between the server and client.
      • HTML Hidden Field
        • We can make unique hidden field in the HTML so users can start to navigate, we can set its value unique to the user and keep track of the session.
      • URL Rewriting
        • We can attach a session identifier parameter in every request and response to keep track of the session.
      • Cookies
        • Cookies are sent by the server with response header and it is being stored in the browser sookies.
        • Everytime a client create requests, it adds the cookie to the request header and we can utilize it to keep track of the session.
      • Session Management API
        • It is built with the other methds for session tracking.
  • Session Handling in Java - Cookies
    • It is used to personalize responses based on the users choice or to keeo in track in the session.
  • Session Handling in Java Servlet - HTTPSession
    • Servlet API gives Session management through the use of HttpSession interface.
    • HttpSession allows us to set objects as attributes that can be retrieved in future requests.
    • Ways to provide an identifier
      • HttpSession getSession()
        • It returns a HTTPSession Object.
        • It returns a session object that is being attached with a request.
      • HttpSession getSession(boolean flag)
        • It returns a HTTPSession Object if a request contains a session otherwise it returns null.
        • Methods of HTTPSession:
          • String getId()
            • it returns a string containing the unique identifier assigned to this session.
          • Object getAttribute(String name)
            • it returns an object bound with the specified name in thie session, or null if no object is bound under the name.
            • Methods to work with Session attributes
              • getAttributeNames()
              • removeAttribute(String name)
              • setAttribute(String name, Object value)
          • long getCreationTime()
            • it returns the time when this session was created which was January 1, 1970.
            • We can get last accessed time with getLastAccessedTime() method.
          • setMaxInactiveInterval(int interval)
            • it identifies the time, in seconds of client's requests before the servlet container will invalidate this session.
            • We can get session timeout value using the method getMaxInactiveInterval().
          • ServletContext getServletContext()
            • it returns the ServletContext object for the application.
          • boolean isNew()
            • it returns true if the client does not yet know about the session or if the client chooses not to join the session.
          • void invalidate()
            • it invalidates a session then unbinds any objects bound to it.
  • Session Handling in Java Servlet - URL Rewriting
    • We can encode URL with HttpServletResponse encodeURL() method and if we have to redirect the request to another resource and we want to provide session information, we can use encodeRedirectURL() method.

Session Handling in PHP

  • PHP is one of the widely used Server side scripting language
  • Session Handling supports in PHP that consists of a way to preserve specific data across subsequent accesses.
  • List of PHP Functions:
    • session_start()
      • It starts new or resumes an existing session.
    • session_isset()
    • session_unset()
      • It sets all session variables to free.
    • session_destroy()
      • It destroys all registered data to a session.
